# 🔧 Oath Gateway 環境變數配置

## 📋 必須配置

### 1. 服務識別
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `SERVICE_NAME` | 服務名稱 | `oath-gateway` | 否 |
| `SERVICE_VERSION` | 服務版本 | `1.0.0` | 否 |

### 2. Site Bridge 連接
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `SITE_BRIDGE_URL` | Site Bridge 服務地址（必須可從公網訪問） | `https://site-bridge.your-domain.com` 或 `http://公网IP:端口` | **是** |
| `SITE_BRIDGE_TIMEOUT` | Site Bridge 超時（秒） | `30` | 否 |

### 3. AI 服務配置
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密鑰 | `sk-xxxxxxxxxxxxxxxx` | **是** |
| `DEEPSEEK_TIMEOUT` | DeepSeek 超時（秒） | `60` | 否 |
| `OPENAI_API_KEY` | OpenAI API 密鑰（備援） | `sk-xxxxxxxxxxxxxxxx` | 否 |
| `OPENAI_TIMEOUT` | OpenAI 超時（秒） | `60` | 否 |

### 4. 模型配置
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `DEFAULT_MODEL` | 默認模型名稱 | `oath-gateway` | 否 |
| `FALLBACK_MODEL` | 備援模型名稱 | `gpt-3.5-turbo` | 否 |

## 📁 可選配置

### 5. 路徑配置
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `PHOTOS_DIR` | 照片存儲目錄（Railway Volume） | `/data/photos` | 否 |
| `RESULTS_DIR` | 結果存儲目錄（Railway Volume） | `/data/results` | 否 |

### 6. RAG 配置
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `RAG_URL` | RAG 服務地址 | `http://rag-service:8000` | 否 |

### 7. 公開 URL 配置
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `OATH_PUBLIC_BASE` | 公開基礎 URL（用於生成下載連結） | `https://oath-gateway.up.railway.app` | 否 |

### 8. 服務器配置
| 變數名稱 | 說明 | 範例值 | 必須 |
|----------|------|--------|------|
| `HOST` | 綁定主機 | `0.0.0.0` | 否 |
| `PORT` | 綁定端口 | `8000` | 否 |
| `DEBUG` | 調試模式 | `false` | 否 |

## 🚀 Railway 環境變數設置

### 方法1：Railway Dashboard
1. 進入 Railway 項目
2. 點擊 "Variables"
3. 添加上述環境變數
4. 點擊 "Save"

### 方法2：Railway CLI
```bash
railway variables set SITE_BRIDGE_URL=http://192.168.1.100:9001
railway variables set DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
railway variables set OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx
```

### 方法3：railway.json（不推薦用於敏感信息）
```json
{
  "variables": {
    "SITE_BRIDGE_URL": "http://192.168.1.100:9001",
    "DEFAULT_MODEL": "oath-gateway"
  }
}
```

## 🔒 安全建議

### 敏感信息保護
1. **API 密鑰**：永遠不要提交到版本控制
2. **環境變數**：使用 Railway 的加密存儲
3. **訪問日誌**：定期檢查訪問日誌

### 最小權限原則
1. 只配置必要的環境變數
2. 使用不同的 API 密鑰用於不同環境
3. 定期輪換 API 密鑰

## 🧪 測試配置

### 配置驗證腳本
```python
import os

required_vars = [
    "SITE_BRIDGE_URL",
    "DEEPSEEK_API_KEY"
]

missing = []
for var in required_vars:
    if not os.getenv(var):
        missing.append(var)

if missing:
    print(f"❌ 缺少必須配置: {missing}")
else:
    print("✅ 配置檢查通過")
```

### 健康檢查
部署後訪問：
```
GET https://your-service.railway.app/health
```

預期響應：
```json
{
  "ok": true,
  "service": "oath-gateway",
  "config_errors": [],
  "site_bridge": {"ok": true}
}
```

## 🔄 環境變數更新

### 更新步驟
1. 在 Railway Dashboard 更新變數
2. 重啟服務：`railway restart`
3. 驗證配置：訪問 `/health` 端點

### 批量更新
```bash
# 從 .env 文件導入
railway variables --import .env.production

# 導出當前配置
railway variables --export > .env.backup
```

## 📝 配置示例

### 完整配置示例（.env.example）
```bash
# 服務配置
SERVICE_NAME=oath-gateway
SERVICE_VERSION=1.0.0
DEBUG=false

# Site Bridge 配置
SITE_BRIDGE_URL=http://192.168.1.100:9001
SITE_BRIDGE_TIMEOUT=30

# AI 服務配置
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
DEEPSEEK_TIMEOUT=60
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx
OPENAI_TIMEOUT=60

# 模型配置
DEFAULT_MODEL=oath-gateway
FALLBACK_MODEL=gpt-3.5-turbo

# 路徑配置
PHOTOS_DIR=/data/photos
RESULTS_DIR=/data/results

# RAG 配置
RAG_URL=http://rag-service:8000

# 公開 URL
OATH_PUBLIC_BASE=https://oath-gateway.up.railway.app

# 服務器配置
HOST=0.0.0.0
PORT=8000
```

### 最小配置示例
```bash
SITE_BRIDGE_URL=http://192.168.1.100:9001
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
```

## 🐛 故障排除

### 常見問題
1. **配置未生效**：重啟服務 `railway restart`
2. **Site Bridge 連接失敗**：檢查網絡連接和防火牆
3. **API 密鑰無效**：驗證密鑰是否正確且未過期

### 診斷命令
```bash
# 檢查環境變數
railway variables

# 查看服務日誌
railway logs

# 檢查服務狀態
railway status
```
